# 🤝 기여 가이드 (Contributing Guide)

ChatGPT 프롬프트 100개 프로젝트에 기여해주셔서 감사합니다! 이 문서는 프로젝트에 기여하는 방법을 안내합니다.

## 🎯 기여 방법

### 1. 새로운 프롬프트 제안
- 실제로 테스트해본 유용한 프롬프트가 있다면 제안해주세요
- [Issues](https://github.com/yourusername/chatgpt-prompts-100/issues)에 "새 프롬프트 제안" 라벨로 등록

### 2. 기존 프롬프트 개선
- 더 효과적인 프롬프트 개선안이 있다면 Pull Request로 제출
- 개선 이유와 테스트 결과를 함께 제공

### 3. 번역 기여
- 다른 언어로의 번역 (일본어, 중국어, 스페인어 등)
- 기존 번역의 오류 수정

### 4. 문서 개선
- README 개선
- 사용법 가이드 보완
- 오타 수정

## 📝 Pull Request 가이드라인

### 브랜치 명명 규칙
```
feature/새기능명
fix/수정내용
docs/문서수정
translation/언어코드
```

### 커밋 메시지 형식
```
[타입] 간단한 설명

상세 설명 (선택사항)

관련 이슈: #이슈번호
```

**타입 예시:**
- `feat`: 새로운 프롬프트 추가
- `fix`: 프롬프트 수정
- `docs`: 문서 수정
- `style`: 포맷팅 수정
- `refactor`: 구조 개선
- `test`: 테스트 추가
- `chore`: 기타 작업

### PR 템플릿
```markdown
## 변경 사항
- [ ] 새 프롬프트 추가
- [ ] 기존 프롬프트 개선
- [ ] 번역 추가/수정
- [ ] 문서 개선
- [ ] 버그 수정

## 설명
변경 사항에 대한 상세 설명을 작성해주세요.

## 테스트 결과
새로운 프롬프트의 경우 실제 테스트 결과를 공유해주세요.

## 관련 이슈
Closes #이슈번호
```

## 🌟 프롬프트 품질 기준

### 좋은 프롬프트의 조건
1. **구체성**: 명확하고 구체적인 지시사항
2. **재사용성**: 다양한 상황에 적용 가능
3. **효과성**: 실제로 유용한 결과 생성
4. **간결성**: 불필요한 내용 없이 간결

### 프롬프트 형식
```markdown
### [번호]. [프롬프트 제목]
```
[설명 또는 사용 맥락]
```

**예시:**
```markdown
### 15. 과제 분석
```
[과제 문제]를 작업하고 있습니다. 지금까지 [작업한 내용]을 완성했습니다. 어디서부터 더 진행해야 할지 아이디어를 도와줄 수 있나요?
```

## 🔍 코드 리뷰 프로세스

### 리뷰 기준
1. **정확성**: 프롬프트가 의도한 결과를 생성하는가?
2. **유용성**: 실제 사용자에게 도움이 되는가?
3. **카테고리**: 적절한 카테고리에 분류되었는가?
4. **형식**: 일관된 형식을 따르는가?

### 승인 조건
- 메인테이너 2명 이상의 승인
- 실제 테스트 완료
- 문서 업데이트 포함

## 🏷️ 이슈 라벨 시스템

- `enhancement`: 기능 개선
- `bug`: 버그 신고
- `documentation`: 문서 관련
- `translation`: 번역 관련
- `new-prompt`: 새 프롬프트 제안
- `good first issue`: 초보자 환영
- `help wanted`: 도움 요청

## 🌍 커뮤니티 가이드라인

### 행동 강령
1. **존중**: 모든 기여자를 존중합니다
2. **건설적**: 건설적인 피드백을 제공합니다
3. **협력**: 함께 더 나은 프로젝트를 만들어갑니다
4. **학습**: 서로에게서 배우는 자세를 유지합니다

### 소통 채널
- **GitHub Issues**: 버그 신고, 기능 제안
- **GitHub Discussions**: 일반적인 논의
- **Discord**: 실시간 채팅 (링크 제공 예정)
